GITJOKER-C2

Layer di Command & Control orientato a publishing e compliance su GitHub: standardizza input/output, esegue controlli ex-ante, produce eventi auditabili.

UE-first Audit-first Fail-closed Hash-only

README Operating Model

Cosa fa

• Controllo ex-ante di release e pubblicazioni
• Standardizzazione di evidenze (manifest, receipt, freeze)
• Produzione di audit trail append-only
• Routing verso moduli IPR/UNEBDO/OPC

Cosa non fa

• Non è uno strumento offensivo / operativo “tattico”
• Non automatizza decisioni legali o istituzionali
• Non opera in modalità permissiva
• Non gestisce dati personali (hash-only)

Regola FAIL-CLOSED

Se un controllo fallisce (hash, firma, policy, integrità repo), la pubblicazione è bloccata. Nessun output “parziale” viene considerato valido.

Collegamenti chiave

• EU Gateway: piattaforma neutra Create/Verify/Evidence
• IPR stack: ipr-core / ipr-gate / ipr-comply / ipr-exchange
• Policy: UNEBDO + OPC
