## Repository Governance (GITJOKER-C2)

Questo repository segue una separazione chiara tra:
- **Vetrina pubblica** (README)
- **Storia operativa** (CHANGELOG)
- **Documentazione di supporto** (docs/)
- **Registri e prove** (registry/)

### Regole
- Il file **README.md** viene aggiornato **solo** quando cambia il messaggio pubblico o il posizionamento del prodotto.
- Le modifiche operative e incrementali vengono registrate in **CHANGELOG.md**.
- Le prove, i riferimenti esterni e i mapping di versione sono inseriti in **registry/**.
- Ogni commit è responsabilità dell’operatore umano che lo esegue.

Questa governance garantisce leggibilità, tracciabilità e verificabilità nel tempo.
