# GITJOKER-C2 — Publishing & Compliance Control (UE-first)

GITJOKER-C2 è il layer di controllo per **pubblicazione** e **compliance** su GitHub
nell’ecosistema Hermeticum B.C.E.

## Principi
UE_FIRST · AUDIT_FIRST · FAIL_CLOSED · HASH_ONLY

## Funzione
- controllo ex-ante delle pubblicazioni (deny-by-default)
- standardizzazione di output verificabili (manifest / receipt / freeze)
- audit trail append-only
- routing verso moduli IPR, UNEBDO, OPC

## FAIL-CLOSED (regola)
Se un controllo fallisce o è indeterminato:
- la pubblicazione è negata
- l’output è invalidato
- l’evento è auditabile

## Non è
- uno strumento offensivo
- un sistema decisionale automatico
- un sostituto della legge o delle autorità
